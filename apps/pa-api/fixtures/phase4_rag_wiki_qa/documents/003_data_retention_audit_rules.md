# 数据留存与访问审计法规条款

锚点：TEST-RAG-003
材料类型：法规法条类
发布日期：2025-01-12
适用范围：虚构资料库、问答记录、引用证据、访问审计和留痕流程
敏感性说明：本材料为合成脱敏测试文本，不对应任何真实法规、真实条款或真实监管要求。

## 第一条 定义

本规则所称“资料留存”，是指对上传文档、解析片段、问答结果、引用清单和人工确认记录进行保存，以便后续检索、复核和问题定位。本规则所称“访问审计”，是指记录谁在何时查看、检索、导出或修改了资料及其引用结果。

本规则不涉及真实个人信息，也不包含真实客户资料。文中“资料库”均指测试环境中的虚构知识库。

## 第二条 留存期限

一般测试资料的留存期限为 **九十日**。若资料被用于知识问答质量评估，可延长至 **一百八十日**，但必须在测试清单中标记用途。若资料属于一次性演示材料，且未进入问题集，可在三十日后删除。

留存期限用于测试条款定位能力。问题若询问“一般测试资料保留多久”，期望命中本条并回答九十日；若询问“质量评估资料保留多久”，期望回答一百八十日。

## 第三条 访问审计字段

访问审计至少记录以下字段：

1. 操作时间。
2. 操作人代号。
3. 操作类型：查看、检索、上传、发布、撤回。
4. 资料锚点或 Wiki 页面锚点。
5. 是否触发下载或复制。
6. 操作结果：成功、失败、被拒绝。

审计记录不得保存完整原文、完整问答 prompt 或密钥信息。若需要定位问题，应保存短片段、锚点和 trace id。

## 第四条 禁止事项

禁止在审计记录中写入 API Key、service token、真实内网地址、未脱敏原文或真实个人标识。禁止将测试资料误标为真实生产资料。禁止使用访问审计记录反向恢复完整文档。

本条容易与 `TEST-RAG-004` 的“外部材料引用发布校验”混淆。区别在于：本规则关注留存和访问审计；`TEST-RAG-004` 关注对外发布前的引用校验。

## 第五条 删除与复核

删除测试资料前，应确认该资料未被当前问题集引用。若资料锚点仍出现在命中矩阵中，应先更新问题集或替换为新的合成材料。删除操作应保留删除原因和执行人代号，但不得保留已删除资料正文。

## 第六条 测试提示

本文关键词包括：九十日、一百八十日、访问审计字段、不得保存完整 prompt、trace id、删除前检查命中矩阵。若问题询问“发布前是否需要人工确认引用”，应命中 `TEST-RAG-004`，不是本文。
