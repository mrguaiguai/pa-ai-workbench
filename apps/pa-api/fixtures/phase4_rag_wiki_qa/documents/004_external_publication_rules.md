# 外部材料引用与发布校验法规条款

锚点：TEST-RAG-004
材料类型：法规法条类
发布日期：2025-01-20
适用范围：虚构外部材料、引用校验、发布确认和撤回流程
敏感性说明：本材料为合成脱敏测试文本，不对应任何真实法规、真实条款或真实监管要求。

## 第一条 目的

本规则用于测试系统对“引用校验”和“发布校验”的理解。它与 `TEST-RAG-003` 同样使用条款结构，但关注点不同：本规则关注材料对外使用前，是否完成来源确认、引用核对、版本检查和撤回预案。

## 第二条 引用校验

所有拟用于外部说明、公开材料或跨团队共享材料的内容，必须完成引用校验。引用校验至少包括：

1. 引用片段是否来自已入库资料或已发布 Wiki。
2. 引用片段是否能定位到文档锚点、chunk id 或 Wiki 页面锚点。
3. 结论是否超出引用片段的表达范围。
4. 是否引用了旧版要求而忽略新版要求。
5. 是否存在无来源的判断性表述。

若任一项无法确认，材料状态应标记为“待补证据”，不得标记为“可发布”。

## 第三条 发布校验

发布前必须由复核人确认以下事项：

- 资料来源已确认。
- 引用清单不包含待复核附件。
- 版本冲突已说明。
- 涉及推断的内容已标记为“基于材料推断”。
- 无答案问题没有被强行生成确定性结论。

发布校验记录应保存复核时间、复核人代号、材料锚点和最终状态。发布校验记录不应保存完整原文或敏感凭据。

## 第四条 撤回规则

若发布后发现引用错误，应在两个工作日内完成撤回说明，并在知识库中标记原材料为“需复核”。撤回说明应写明错误引用位置、正确引用来源、影响范围和后续处理人。若错误仅影响内部测试，不需要对外撤回，但仍需在测试记录中保留更正原因。

## 第五条 与访问审计的区别

访问审计记录“谁做了什么操作”，见 `TEST-RAG-003`。发布校验记录“材料是否可以对外使用”。二者都强调不得保存密钥和完整原文，但用途不同。测试问题若询问“发布前要检查哪些引用事项”，应命中本文；若询问“审计字段有哪些”，应命中 `TEST-RAG-003`。

## 第六条 测试提示

本文关键词包括：引用校验、待补证据、可发布、版本冲突、撤回说明、两个工作日、发布校验。它用于测试条款定位、相似条款区分和无来源结论拒答。
