# PA AI Workbench documentation

This tree separates current product truth, current acceptance evidence, and
historical stage records.

- [`product/`](product/README.md) — product-facing references and terminology.
- [`architecture/`](architecture/README.md) — current ownership and capability
  architecture.
- [`operations/`](operations/README.md) — current root commands and runbooks.
- [`stages/current/`](stages/current/README.md) — the active PAR stage contract.
- [`evidence/`](evidence/README.md) — current PAR task evidence.
- [`handoff/`](handoff/README.md) — final handoff boundary; P4 evidence is not
  pre-created.
- [`archive/`](archive/README.md) — completed historical stages and legacy
  product specifications.

Archived reports describe their original run. Re-run the canonical validation
commands before making a claim about the current checkout.
